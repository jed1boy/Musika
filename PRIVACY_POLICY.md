# Privacy Policy for Musika

Last updated: 2026-02-25

## Overview

Musika is designed to be privacy-focused.  
We do not use Firebase Analytics, Firebase Crashlytics, or ad trackers.

Most data is stored locally on your device and is only used to provide core app features.

## Data We Store Locally

Musika may store the following on your device:

- App settings (theme, language, playback preferences)
- Search history (if enabled)
- Listen history (if enabled)
- Playlists and library metadata
- Account data needed for YouTube Music login sessions
- Optional proxy settings
- Optional AI translation settings

Sensitive preferences (for example, login/session values and API keys) are stored in encrypted local storage.

## Permissions

Musika requests permissions only for features you use:

- Microphone: song recognition
- Notifications: playback controls and update notifications
- Bluetooth / Nearby Wi-Fi / location (older Android): cast and nearby device discovery
- Media storage access: local audio playback

You can revoke permissions at any time in Android system settings.

## Network Requests

Musika connects to third-party services only to deliver requested features, such as:

- YouTube Music (playback, account/library sync)
- Lyrics providers
- SponsorBlock
- Shazam
- OpenRouter (only if you enable AI translation and provide an API key)
- GitHub (update checks)

Musika does not sell your personal data.

## Local Network Features

If you use Cast or DLNA features, Musika may discover devices on your local network and send playback metadata to selected devices.

If you do not use these features, this traffic does not occur.

## Privacy Controls in App

Musika includes privacy controls such as:

- Pause listen history
- Pause search history
- Incognito mode
- Clear listen/search history
- Auto-clear history on app close
- Disable screenshots

## Backups

Sensitive app data is excluded from Android backup and device transfer.
